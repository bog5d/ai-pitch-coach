# 系统稳健性检视摘要（2026-04-17）

供外发版接手与主理人自检。**非**否定当前架构，而是标出「已知边界」与「建议演进顺序」。

---

## 1. 已相对稳健的部分

- **契约与回归**：Pydantic `schema` + **637** 单测；重大重构（LangGraph、脱敏、GitHub 同步）均有对应测试。
- **双栈评估**：`USE_LANGGRAPH_V1` / `PitchFileJobParams.use_langgraph_v1` 可回退旧 `evaluate_pitch`。
- **coach_data 上传**：analytics 路径 ASCII 安全化；**analytics / institutions** 分通道状态；锁定后 analytics 不再被「无 institution_id」误跳过。
- **显式拉取**：`pull_analytics_for_company` + UI，避免误认「上云即自动下云」。
- **脱敏**：`sanitize_inputs` + 正则默认、Presidio 可选，兼顾 CI 与合规开关。

---

## 2. 仍需谨慎的薄弱点（按优先级）

### P1 — 多租户与数据隔离（逻辑层）

- **Episodic**（错题本、归档路径）依赖工程约定与 `company_id`；**非**数据库级 ACL。
- **coach_data** 上 institutions 为全局表；analytics 按 segment 分桶，但 **PAT 持有者仍可读全库**——私有部署或分支仓库前需有预期。
- **`.coach_data_pull/`** 与正式归档目录混在同一 workspace 扫描内；长期可考虑「仅扫描白名单子目录」降低误读风险。

### P2 — 运维可观测性

- **`github_sync_status.json`** 已建议 **不入库**（`.gitignore`），仅本机可写根生成；Dashboard 依赖该文件时，新 clone 需先产生一次同步才有状态。
- **锁定失败 / 网络超时**：多数路径静默或 toast；关键路径已有日志，但缺少「用户可导出的同步审计 CSV」类产物（可选演进）。

### P3 — 前端与状态复杂度

- **`app.py` 体量巨大**：Streamlit `session_state` 键多，并行改 UI 易踩 widget 规则（见 `AGENTS.md` 狙击表约束）。
- **发言人映射**：当前主路径偏「首条录音 / batch_iv_0」；多文件批量时映射一致性需后续产品化。

### P4 — 外部依赖

- ASR / LLM 多供应商：**费用与可用性**依赖环境变量；无统一「降级模式开关」UI（仅代码与环境层）。

### P5 — 仓库卫生（开发者）

- 根目录下历史 **ZIP / 本地归档** 不应提交；**`.env` 已忽略**；大文件音频/HTML 已在 `.gitignore`。

---

## 3. 外发版（纯净交付包）包含什么

- 以 `build_release.py` → `CURRENT_VERSION` 为准；默认 **`src/` + 根必需文件 + 可选说明 txt + `AGENTS.md` + `docs/strategic/*.md`**。
- **不包含**：`.env`（仅空文件占位）、**不含** 用户录音与报告、**不含** `.executive_memory` / `.drafts` 等运行期数据。
