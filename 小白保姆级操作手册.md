## 1️⃣ 欢迎致辞与系统简介

嗨，欢迎你 👋

从现在开始，你手里多了一个「**AI 路演与访谈复盘系统**」——不用装专业剪辑软件，也不用自己熬夜听写。

### 一句话看懂价值

> **它能把又长又枯燥的录音，全自动变成一份「绝美网页报告」**：  
> 带 **音频高亮切片**、带 **毒舌式逻辑点评**，还能把你上传的 **内部 QA 材料** 对齐到发言里——**一屏看完、一点就听**，像给老板做「路演复盘 PPT」一样体面。

## 2️⃣ 界面区域「大解剖」——这是干嘛的？

下面每一块，都对应 `app.py` 里真实存在的界面，照着认就不会迷路。

### 🔑 左侧栏 · 密钥配置（`🔑 首次使用请配置 API 密钥`）

| 你在界面上看到的 | 它在干什么？ |
| :--- | :--- |
| **阿里云 API Key（用于转写）** | 交给云端「**神仙耳朵**」——把录音变成带时间戳的文字，是整条流水线的**第一步**。 |
| **DeepSeek API Key（用于毒舌分析）** | 交给「**毒舌大脑**」——在转写文本的基础上，按业务场景做**找茬式复盘**，输出结构化报告。 |
| **💾 保存并测试连接** | 把 Key **写进本地 `.env` 并当场测连通性**，两边都绿灯才算真正可用。 |

**底层逻辑（一句话）：**  
转写 + 打分 = **两条不同的云服务**，所以要两把 Key；**填完点一次「保存并测试」**，全绿才能点主按钮——**一般只需填一次**，以后打开网页会自动记住（在本地配置文件里）。

---

### 📁 左侧栏 · 数据归档根目录（`数据归档根目录 (Workspace Root)`）

把它想象成公司的 **「大仓库」入口**：

- 系统跑完后，会在你选的根目录下面，**自动按「业务大类 → 项目/批次」建好文件夹**。
- 录音、转写 JSON、分析报告、复盘 HTML 等，都会 **分门别类** 落在这个仓库里。

> 💡 **小白理解：** 这里不是「上传文件」的地方，而是**告诉系统：把成品快递寄到哪个仓库**。

---

### 🏷️ 主界面 · 业务大类 & 项目/批次名称

| 控件 | 作用 |
| :--- | :--- |
| **业务大类**（下拉框） | 例如 `01_机构路演`、`02_高管访谈` … 系统会把它写进 **Prompt 与文件夹名**。 |
| **项目/批次名称** | 例如 `福创投`、某次尽调批次名 —— 既是**子文件夹名**，也是给 AI 的 **「项目代号」**。 |

**重要：** 这不只是「建文件夹的名字」那么简单，更是 **给 AI 大脑洗脑的「背景设定」**：

- 你选 **「高管访谈」**，AI 就会用 **「高管视角」** 去听——语气、责任、表述是否经得起推敲。
- 你选 **「客户访谈」**，AI 就会更关注 **客户侧叙事与事实**。

> 选什么场景，就等于给 AI **戴上一副什么样的眼镜** 👓

---

### 📚 主界面 · 参考 QA（Tab：`参考 QA 说明`）

- 这个 Tab 里是 **说明文字**：真正上传 QA 的位置在下面的 **「逐录音填写」**。
- **每一条录音** 都有自己那一块的 **「本段参考 QA（可选，可多选）」**——把和 **这一段访谈** 对口的口径、清单、标准答案丢进去即可。
- **只录了一段音** 也一样：上传音频后，只会出现 **文件 1** 那一块，在那里面传 QA。
- 系统把 QA 当成 **「开卷考试的标准答案」**；**不传也能生成报告**，但对齐深度通常会弱一些。

**支持格式（请优先记这三类）：**

| ✅ 支持 | 说明 |
| :--- | :--- |
| **Word（.docx）** | 制度、流程、口径说明 |
| **PDF（.pdf）** | 财报摘要、对口径 PDF 等 |
| **Excel（.xlsx）** | 表格、指标表 |

> 另：**txt / md** 纯文本也可上传（适合备忘录/纪要）；**日常业务文档仍以 Word / PDF / Excel 为主。**

> **❌ 不支持 PPT（.ppt / .pptx）**  
> —— **PPT 结构复杂、排版多变**，直接解析容易 **乱码或导致系统崩溃**；为保证稳定，**请把 PPT「另存为 PDF」再上传**。

---

### 🎵 主界面 · 上传录音（`上传音频（可多选）`）

- **支持多选**：一次丢进去多个录音也可以；**只丢一个也完全没问题**（和多条是同一套操作）。
- 常见格式：**m4a / mp3 / wav / mp4 / webm** 等（以界面提示为准）。

---

### 📝 主界面 · 逐录音填写（`逐录音填写（进入 AI 上下文）`）

上传音频后才会出现这一块。每一段录音对应：

| 区域 | 说明 |
| :--- | :--- |
| **根据录音文件名自动填写…**（勾选框，默认开） | 若文件名像 **`机构-姓名20260108`**，会试着自动填 **被访谈人** 和下面那一栏；不对就手改。 |
| **被访谈人（必填）** | 写清楚这一段是谁在说。 |
| **结构化狙击清单（原文引用 / 找茬疑点，可选）** | 表格里一行一对：**原文引用**贴转写原话，**找茬疑点**写要找什么茬、核对什么口径。可点加号多行；表格内容绑在会话里，一般交互重绘不易丢。**字尽量从「仅提取文字稿」里抄**（按 **[发言人 1]**、**[发言人 2]** 分段，好对准），不要凭记忆瞎写。不填也能跑完全流程。 |
| **本段参考 QA（可选）** | 这一段要对照的内部材料（Word/PDF 等）；多份会先合在一起，再取前大约 **3 万字** 给 AI 看。 |

> 若你 **调整过音频顺序或增删文件**，请逐条核对：人、备注、QA 是否还和 **当前这一条录音** 对得上。

### 其它你迟早会看到的（左侧栏）

| 区块 | 说明 |
| :--- | :--- |
| **🔒 敏感词汇黑名单** | 逗号分隔多个词。系统会在**发给大模型之前**把词打成 `***`，**保护公司机密**（详见第 5 节）。 |
| **恢复上次未完成的审查草稿**（有时出现） | 若你上次改到一半关了网页、或断网了，**再次打开**且当前没有在改的审查页时，左边可能出现这句话。**点一下**就能把上次的改动能找回来（存在你自己电脑/服务器上的隐藏小文件夹里，不叫上传云端）。正在正常改审查台时一般不会打扰你。 |
| **主按钮「开始生成复盘报告」** | 只有 **阿里云、DeepSeek、FFmpeg 三项全绿**（与侧栏「保存并测试连接」结果一致）后才会亮起可点；否则是灰色，防止「没油还开车」。 |

### 📝 主按钮上面：先拿文字稿（可选，但很管用）

在点 **「开始生成复盘报告」** 之前，你会看到一个按钮，大意是 **「仅提取文字稿……可复制去做精准核实」**：

- **干什么用**：先把这条（或列表里**第一条**）录音**只转成文字**，显示在大框里，**你可以整段复制**。
- **为什么要用**：填 **原文引用** 列时，**直接粘贴「仅提取文字稿」里对应说话人的原话**，AI 更容易对准你想听的那一段，少「张冠李戴」。
- **注意**：多段录音时，这个按钮**默认只转写第一段**；要转写别的，可先只上传那一条，或请同事按你们流程操作。

## 3️⃣ 手把手实战演练（跟着我做一次）

### 🎬 场景设定（虚构但非常真实）

> 刚结束 **福创投** 对你们公司 **研发孙总** 的 **高管访谈**。  
> 你手里有：  
> - 录音：`孙总访谈.m4a`  
> - 口径：`公司财务与业务口径.pdf`（最新版）

---

### Step 1：打开网页，先「左边亮绿灯」

1. 用浏览器打开系统（按你们 IT 给的启动方式或 `streamlit` 地址）。
2. 看 **左侧侧边栏** → 展开 **「🔑 首次使用请配置 API 密钥」**。
3. 填好 **阿里云 Key** + **DeepSeek Key**，点 **「💾 保存并测试连接」**（会依次检测阿里云、DeepSeek，并探测本地 **FFmpeg**）。
4. 看到 **三项均通过** 与 **「✅ 环境：API + FFmpeg 已验证」**（与当前 `app.py` 文案一致）后，再往下走。

---

### Step 2：选对「大脑设定」

1. **业务大类** 下拉框 → 选 **`02_高管访谈`**（别选错，这是给 AI 的「眼镜」）。
2. **项目/批次名称** → 填 **`福创投`**（和真实项目名一致即可）。

---

### Step 3：上传录音

1. 在 **「上传音频（可多选）」** 里，拖入 **`孙总访谈.m4a`**。

---

### Step 4：填这一段的人 + 传这一段用的 QA

1. 页面上会出现 **「逐录音填写」**：这里是 **文件 1：`孙总访谈.m4a`**。
2. 看 **被访谈人** 是否已经根据文件名填好；不对就改成 **「研发孙总」** 或你们内部习惯写法。
3. **本段参考 QA** 里拖入 **`公司财务与业务口径.pdf`**（可多选；和这段访谈无关的材料别混进来）。
4. （可选）看一眼 Tab **「参考 QA 说明」** 里的格式说明，心里有个数即可。
5. （**可选，想盯某句话时很有用**）本条的人名、QA 都填好后，**往页面下面稍微滚一点**（在点「开始生成」之前），点 **「仅提取文字稿」**，等按说话人分段的字出来后，把相关句子 **复制** 到本条 **原文引用** 列（**找茬疑点**写清要核实什么），再点生成。

---

### Step 5：点击生成，然后去泡杯咖啡 ☕

1. 点主按钮 **「开始生成复盘报告」**。
2. 页面上会展开一块 **进度说明**（灰色可展开区域），大意包括（以你屏幕上为准）：
   - 收到了多大的文件；若文件特别大，可能会先 **压一压再送去听写**；  
   - 然后走 **听写 → 敏感词打码（你设的黑名单会变 `***`）→ AI 先在网页里写一版初稿**；  
   - 最后告诉你：**初稿先进审查台**，不会立刻生成最终网页报告。  
   **最费时间的一般是听写和 AI 那两步**，别急着刷新，等它跑完。
3. 全部成功时，页面可能会 **冒气球** 🎈，并出现 **【🔍 报告审查与人工编辑台】**——**请先在这里校对，再导出**（见下一节）。

---

### Step 5.5：审查台里干什么？（V3.0 必看）

跑完后页面会出现 **【报告审查与人工编辑台】**，按 **每一条录音** 分页查看：

| 你能做的事 | 说明 |
| :--- | :--- |
| **改 AI 评语** | 各维度打分说明、扣分理由等可在文本框里直接改。 |
| **删片段** | 某条「翻车」若不合理，点 **🗑️ 删除此片段** 会从当前草稿里去掉。 |
| **听 + 对稿** | 片段下方可 **试听** 一小段录音；旁边框里是 AI 先写的一版说明，方便你对照。**你点「锁定」后，真正发出去的网页里，「发言人口述实录」会再按录音里的原话自动对齐一遍**，避免和内部参考材料「串台」。**若框里和网页里不一致，以网页里为准。** |
| **人工补刀** | 点 **➕ 添加人工发现的复盘点**，可手写标题、问题、改进建议。 |
| **定稿** | 全部满意后，点该录音下方的 **【✅ 确认无误，锁定并生成最终版 HTML 报告】**。  
| **只有点了「锁定」** | 系统才会 **覆盖保存** 对应的 `*_analysis_report.json`，并生成 **`*_复盘报告.html`**。 |

> **刷新网页**：若没再点「开始生成」，但上次已经跑出过审查台，页面仍会显示审查台，可继续改到满意再锁定。  
> **重新跑一批**：再次点「开始生成」会 **清空当前审查草稿**，以新一轮结果为准。  
> **悄悄自动存盘**：只要审查台还在，系统会**时不时把你改的东西存到本机一份**；关浏览器太猛时，下次可从左边 **「恢复草稿」** 试试（见上文侧栏说明）。

## 5️⃣ 避坑指南（注意事项）

| ⚠️ 注意点 | 说明 |
| :--- | :--- |
| **录音格式** | 优先 **m4a / mp3 / wav**，兼容性好、转写更稳。 |
| **长录音** | 例如 **1 小时** 量级，听写 + AI 可能要 **好几分钟**。**正在转圈、进度条在动的时候别刷新**，容易前功尽弃。（全部跑完后，你再慢慢改审查台，偶尔刷新一般还能接着改，见 Step 5.5） |
| **忘了点「锁定」** | 没有 HTML 或 JSON 还是初稿：回到页面若仍显示审查台，改完点 **锁定**；若已清空，需重新跑生成。 |
| **敏感词** | 涉及 **极度机密财务数据、客户名、内部代号** 等，请在左侧 **「🔒 敏感词汇黑名单」** 里用英文逗号分隔写上，例如：`净利润, 某客户代号`。系统会在送大模型前打成 **`***`**。 |
| **PPT** | 不支持直接上传；**请另存为 PDF** 再上传。 |
| **QA 太长** | 合并后超过 UI 截断上限时，会先截到约 **3 万字**；若仍超过模型侧 QA 池，会**智能保留头尾**并**黄字提示**——**把最重要的条款尽量放在文档前后**更划算。 |

## 6️⃣ 多人共享网盘配置（V10.0 新功能）

> 如果你们团队有多台电脑，希望**共用同一套记忆库和转写缓存**，这节就是给你的。

### 核心原理

默认情况下，系统把记忆库和转写缓存存在本机的 `.executive_memory` 和 `.asr_cache` 文件夹。

配置共享盘后，全组的记忆和缓存都写到**共享盘同一目录**，A 同事跑完的录音，B 同事不用重新转写。

### 配置步骤（以 Windows + 123 网盘为例）

**第一步：挂载网盘为本地盘符**

1. 打开 123 网盘客户端，在设置里找"挂载为本地磁盘"（通常会分配一个盘符如 `Z:`）。
2. 确认在资源管理器里能看到 `Z:\` 这个盘。

**第二步：在共享盘创建目录**

```
Z:\AI路演教练\executive_memory\   ← 记忆库（错题本）
Z:\AI路演教练\asr_cache\          ← ASR 转写缓存
```

**第三步：配置环境变量**（二选一，推荐方式一）

**方式一：写 `.env` 文件**（单机生效，适合开发调试）

找到项目根目录的 `.env` 文件（没有就从 `.env.example` 复制），加入：

```ini
MEMORY_ROOT=Z:\AI路演教练\executive_memory
CACHE_ROOT=Z:\AI路演教练\asr_cache
```

**方式二：设置 Windows 系统环境变量**（开机自动生效，推荐多人场景）

1. 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
2. 在"系统变量"区点击"新建"：
   - 变量名：`MEMORY_ROOT`，变量值：`Z:\AI路演教练\executive_memory`
   - 变量名：`CACHE_ROOT`，变量值：`Z:\AI路演教练\asr_cache`
3. 确定后**重启程序**生效。

**第四步：验证**

重新打开 AI 路演教练，去**"机构画像看板"**，滚到底部"行业基准对比"区域，正常情况下会显示共享盘里已有的历史分析数据。

### 常见问题

| 问题 | 解答 |
| :--- | :--- |
| 网盘没挂载，程序报错吗？ | 不会。程序会自动回退本机路径，不影响使用。 |
| 两个人同时跑，会冲突吗？ | 记忆库和缓存都是**原子写入**（先写临时文件，写完再移过去），同一目录多人同时写不会损坏数据。 |
| 换了电脑，以前的记忆还在吗？ | 只要网盘还挂着，记忆全部保留，无缝切换。 |
| 有没有速度影响？ | 若网盘网速慢，首次读写可能稍慢；后续 ASR 缓存命中后，跟本机一样快。 |
