## 1️⃣ 欢迎致辞与系统简介

嗨，欢迎你 👋

从现在开始，你手里多了一个「**AI 路演与访谈复盘系统**」——不用装专业剪辑软件，也不用自己熬夜听写。

### 一句话看懂价值

> **它能把又长又枯燥的录音，全自动变成一份「绝美网页报告」**：  
> 带 **音频高亮切片**、带 **毒舌式逻辑点评**，还能把你上传的 **内部 QA 材料** 对齐到发言里——**一屏看完、一点就听**，像给老板做「路演复盘 PPT」一样体面。

## 2️⃣ 界面区域「大解剖」——这是干嘛的？

下面每一块，都对应 `app.py` 里真实存在的界面，照着认就不会迷路。

### 🔑 左侧栏 · 密钥配置（`🔑 首次使用请配置 API 密钥`）

| 你在界面上看到的 | 它在干什么？ |
| :--- | :--- |
| **阿里云 API Key（用于转写）** | 交给云端「**神仙耳朵**」——把录音变成带时间戳的文字，是整条流水线的**第一步**。 |
| **DeepSeek API Key（用于毒舌分析）** | 交给「**毒舌大脑**」——在转写文本的基础上，按业务场景做**找茬式复盘**，输出结构化报告。 |
| **💾 保存并测试连接** | 把 Key **写进本地 `.env` 并当场测连通性**，两边都绿灯才算真正可用。 |

**底层逻辑（一句话）：**  
转写 + 打分 = **两条不同的云服务**，所以要两把 Key；**填完点一次「保存并测试」**，全绿才能点主按钮——**一般只需填一次**，以后打开网页会自动记住（在本地配置文件里）。

---

### 📁 左侧栏 · 数据归档根目录（`数据归档根目录 (Workspace Root)`）

把它想象成公司的 **「大仓库」入口**：

- 系统跑完后，会在你选的根目录下面，**自动按「业务大类 → 项目/批次」建好文件夹**。
- 录音、转写 JSON、分析报告、复盘 HTML 等，都会 **分门别类** 落在这个仓库里。

> 💡 **小白理解：** 这里不是「上传文件」的地方，而是**告诉系统：把成品快递寄到哪个仓库**。

---

### 🏷️ 主界面 · 业务大类 & 项目/批次名称

| 控件 | 作用 |
| :--- | :--- |
| **业务大类**（下拉框） | 例如 `01_机构路演`、`02_高管访谈` … 系统会把它写进 **Prompt 与文件夹名**。 |
| **项目/批次名称** | 例如 `福创投`、某次尽调批次名 —— 既是**子文件夹名**，也是给 AI 的 **「项目代号」**。 |

**重要：** 这不只是「建文件夹的名字」那么简单，更是 **给 AI 大脑洗脑的「背景设定」**：

- 你选 **「高管访谈」**，AI 就会用 **「高管视角」** 去听——语气、责任、表述是否经得起推敲。
- 你选 **「客户访谈」**，AI 就会更关注 **客户侧叙事与事实**。

> 选什么场景，就等于给 AI **戴上一副什么样的眼镜** 👓

---

### 📚 主界面 · 参考 QA（Tab：`参考 QA 说明`）

- 这个 Tab 里是 **说明文字**：真正上传 QA 的位置在下面的 **「逐录音填写」**。
- **每一条录音** 都有自己那一块的 **「本段参考 QA（可选，可多选）」**——把和 **这一段访谈** 对口的口径、清单、标准答案丢进去即可。
- **只录了一段音** 也一样：上传音频后，只会出现 **文件 1** 那一块，在那里面传 QA。
- 系统把 QA 当成 **「开卷考试的标准答案」**；**不传也能生成报告**，但对齐深度通常会弱一些。

**支持格式（请优先记这三类）：**

| ✅ 支持 | 说明 |
| :--- | :--- |
| **Word（.docx）** | 制度、流程、口径说明 |
| **PDF（.pdf）** | 财报摘要、对口径 PDF 等 |
| **Excel（.xlsx）** | 表格、指标表 |

> 另：**txt / md** 纯文本也可上传（适合备忘录/纪要）；**日常业务文档仍以 Word / PDF / Excel 为主。**

> **❌ 不支持 PPT（.ppt / .pptx）**  
> —— **PPT 结构复杂、排版多变**，直接解析容易 **乱码或导致系统崩溃**；为保证稳定，**请把 PPT「另存为 PDF」再上传**。

---

### 🎵 主界面 · 上传录音（`上传音频（可多选）`）

- **支持多选**：一次丢进去多个录音也可以；**只丢一个也完全没问题**（和多条是同一套操作）。
- 常见格式：**m4a / mp3 / wav / mp4 / webm** 等（以界面提示为准）。

---

### 📝 主界面 · 逐录音填写（`逐录音填写（进入 AI 上下文）`）

上传音频后才会出现这一块。每一段录音对应：

| 区域 | 说明 |
| :--- | :--- |
| **根据录音文件名自动填写…**（勾选框，默认开） | 若文件名像 **`机构-姓名20260108`**，会试着自动填 **被访谈人** 和 **本段备注**；不对就手改。 |
| **被访谈人（必填）** | 写清楚这一段是谁在说。 |
| **本段备注（可选）** | 角色、场次、关注点等补充。 |
| **本段参考 QA（可选）** | 这一段专用的口径材料；多文件会先合并再截取前约 15000 字。 |

> 若你 **调整过音频顺序或增删文件**，请逐条核对：人、备注、QA 是否还和 **当前这一条录音** 对得上。

---

### 其它你迟早会看到的（左侧栏）

| 区块 | 说明 |
| :--- | :--- |
| **🔒 敏感词汇黑名单** | 逗号分隔多个词。系统会在**发给大模型之前**把词打成 `***`，**保护公司机密**（详见第 5 节）。 |
| **主按钮「开始生成复盘报告」** | 只有 **阿里云、DeepSeek、FFmpeg 三项全绿**（与侧栏「保存并测试连接」结果一致）后才会亮起可点；否则是灰色，防止「没油还开车」。 |

## 3️⃣ 手把手实战演练（跟着我做一次）

### 🎬 场景设定（虚构但非常真实）

> 刚结束 **福创投** 对你们公司 **研发孙总** 的 **高管访谈**。  
> 你手里有：  
> - 录音：`孙总访谈.m4a`  
> - 口径：`公司财务与业务口径.pdf`（最新版）

---

### Step 1：打开网页，先「左边亮绿灯」

1. 用浏览器打开系统（按你们 IT 给的启动方式或 `streamlit` 地址）。
2. 看 **左侧侧边栏** → 展开 **「🔑 首次使用请配置 API 密钥」**。
3. 填好 **阿里云 Key** + **DeepSeek Key**，点 **「💾 保存并测试连接」**（会依次检测阿里云、DeepSeek，并探测本地 **FFmpeg**）。
4. 看到 **三项均通过** 与 **「✅ 环境：API + FFmpeg 已验证」**（与当前 `app.py` 文案一致）后，再往下走。

---

### Step 2：选对「大脑设定」

1. **业务大类** 下拉框 → 选 **`02_高管访谈`**（别选错，这是给 AI 的「眼镜」）。
2. **项目/批次名称** → 填 **`福创投`**（和真实项目名一致即可）。

---

### Step 3：上传录音

1. 在 **「上传音频（可多选）」** 里，拖入 **`孙总访谈.m4a`**。

---

### Step 4：填这一段的人 + 传这一段用的 QA

1. 页面上会出现 **「逐录音填写」**：这里是 **文件 1：`孙总访谈.m4a`**。
2. 看 **被访谈人** 是否已经根据文件名填好；不对就改成 **「研发孙总」** 或你们内部习惯写法。
3. **本段参考 QA** 里拖入 **`公司财务与业务口径.pdf`**（可多选；和这段访谈无关的材料别混进来）。
4. （可选）看一眼 Tab **「参考 QA 说明」** 里的格式说明，心里有个数即可。

---

### Step 5：点击生成，然后去泡杯咖啡 ☕

1. 点主按钮 **「开始生成复盘报告」**。
2. 你会看到一条 **幽默进度条**（和真实代码一致）：
   - 🎧 **竖起耳朵听**（音频转写中，请耐心喝口水）  
   - 🕵️ **听写完毕** → 脱敏打码（敏感词变 `***`）  
   - 🧠 **顶级 VC 大脑找茬**（最耗时的一步，让子弹飞一会儿）  
   - （V3.0）首轮 **不再自动装订最终 HTML**，而是先给你一份 **可编辑的初稿**（数据在网页里，方便你改完再定稿）
3. 全部成功时，页面会 **放飞气球** 🎈，并出现 **【🔍 报告审查与人工编辑台】**——**请先在这里校对，再导出**（见下一节）。

---

### Step 5.5：审查台里干什么？（V3.0 必看）

跑完后页面会出现 **【报告审查与人工编辑台】**，按 **每一条录音** 分页查看：

| 你能做的事 | 说明 |
| :--- | :--- |
| **改 AI 评语** | 各维度打分说明、扣分理由等可在文本框里直接改。 |
| **删片段** | 某条「翻车」若不合理，点 **🗑️ 删除此片段** 会从当前草稿里去掉。 |
| **听 + 对稿** | 片段下方可 **试听** 对应切片，紧挨着会显示 **本段词级逐字稿**，方便核对。 |
| **人工补刀** | 点 **➕ 添加人工发现的复盘点**，可手写标题、问题、改进建议。 |
| **定稿** | 全部满意后，点该录音下方的 **【✅ 确认无误，锁定并生成最终版 HTML 报告】**。  
| **只有点了「锁定」** | 系统才会 **覆盖保存** 对应的 `*_analysis_report.json`，并生成 **`*_复盘报告.html`**。 |

> **刷新网页**：若没再点「开始生成」，但上次已经跑出过审查台，页面仍会显示审查台，可继续改到满意再锁定。  
> **重新跑一批**：再次点「开始生成」会 **清空当前审查草稿**，以新一轮结果为准。

## 4️⃣ 去哪里验收成果？

### 成品放在哪？

- 去 **你自己设置的「数据归档根目录」**（左侧栏那个「大仓库」路径）。
- 本次示例会落在：

```text
<你的归档根目录>/02_高管访谈/福创投/
```

### 里面有什么？

| 文件（示例） | 是什么？ |
| :--- | :--- |
| `孙总访谈.m4a` | 你上传的原始录音（备份） |
| `孙总访谈_transcription.json` | 带词级时间戳的转写结果 |
| `孙总访谈_analysis_report.json` | 结构化复盘数据（**锁定前**可能是 AI 初稿；**锁定后**与你在审查台里改的一致） |
| **`孙总访谈_复盘报告.html`** | **终极武器：离线可打开的 HTML 报告**（**仅在审查台点击「锁定」后才会生成/更新**） |

### 怎么发给领导 / 同事？

**在审查台锁定后**，把 **`孙总访谈_复盘报告.html`** 拖进微信发出去即可。

- **不用联网、不用装 Python**（浏览器打开即可）  
- **手机点开就能听、能看**（以实际浏览器能力为准）

> 这就是你加班的「交付物」——从「一坨录音」到「一页能汇报的复盘报告」。

## 5️⃣ 避坑指南（注意事项）

| ⚠️ 注意点 | 说明 |
| :--- | :--- |
| **录音格式** | 优先 **m4a / mp3 / wav**，兼容性好、转写更稳。 |
| **长录音** | 例如 **1 小时** 量级，AI 思考 + 转写可能要 **好几分钟**。**千万别在跑流水线时刷新网页**，耐心等进度条跑完。（跑完后想慢慢改审查台，刷新仍可继续，见 Step 5.5） |
| **忘了点「锁定」** | 没有 HTML 或 JSON 还是初稿：回到页面若仍显示审查台，改完点 **锁定**；若已清空，需重新跑生成。 |
| **敏感词** | 涉及 **极度机密财务数据、客户名、内部代号** 等，请在左侧 **「🔒 敏感词汇黑名单」** 里用英文逗号分隔写上，例如：`净利润, 某客户代号`。系统会在送大模型前打成 **`***`**。 |
| **PPT** | 不支持直接上传；**请另存为 PDF** 再上传。 |
| **QA 太长** | 系统会**只读取前约 15000 字**作为参考，避免撑爆模型 Token——**把最重要的几页放前面**更划算。 |
